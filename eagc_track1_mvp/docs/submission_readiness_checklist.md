# Submission Readiness Checklist

This checklist tracks the current local MVP state. It is not an official EAGC submission confirmation.

## Runtime And Container

- Docker container: Dockerfile available for local agent packaging; final organizer submission shape remains pending.
- Official EAGC runtime/API: not integrated yet.
- ProcTHOR/AI2-THOR: not integrated into the stable path.
- Local Qwen vLLM endpoint: expected at `http://127.0.0.1:8000/v1`.

## Core Artifacts

- `world_model.json`: available from demo and smoke runs.
- `episode_log.jsonl`: available from demo and smoke runs.
- `run_audit.json`: available from isolated run directories.
- `track1_score.json`: available for LocalSim Track 1 procedure demos.
- `visual_task_result.json`: available for visual-local hybrid evidence demos.
- Technical report draft: available under `reports/`.
- Demo commands: available in `docs/demo_commands.md`.
- Demo snapshot: generated by `python tools/create_demo_snapshot.py`.
- Clean source package reproducibility check: generated and run with `python tools/check_source_package_repro.py --zip-path dist/eagc_track1_mvp_source.zip`.
- Docker context check: run with `python tools/check_docker_context.py`.

## Training Resource Disclosure

- Current training status: no training and no fine-tuning.
- Current model use: local inference with Qwen3.6-35B-A3B-NVFP4 through vLLM.
- Local model name in config: `qwen3.6-35b-nvfp4`.
- The visual-local hybrid path uses Qwen vision inference only for observation extraction.

## External Dependencies

- Python 3.10+.
- `requests`, `PyYAML`, and standard-library modules for the core MVP.
- Local OpenAI-compatible vLLM endpoint for real Qwen runs.
- Local image files under `assets/test_sequences/bedroom_sequence/` for visual sequence smoke tests.
- AI2-THOR code remains experimental and is not part of the stable gate.

## Hardware Used

- GPU: RTX 5090 32GB.
- Host environment: Windows local development environment.

## Known Limitations

- LocalSim is a self-built development and evaluation environment, not the official hidden evaluation.
- `local_heuristic_score` is a local debugging metric, not an official score.
- Visual-local hybrid tasks are symbolic world-model queries, not physical robot interaction.
- The system does not claim physical actions succeeded in visual-only mode.
- Multi-frame vision uses local static images, not ProcTHOR/AI2-THOR simulation.
- Relation tasks are intentionally conservative: missing or stale active relations produce `uncertain`.
- No model training, distillation, or benchmark optimization has been performed.

## Source Package Reproducibility

Recommended pre-submission source package check:

```powershell
python tools/package_source.py
python tools/check_source_package_repro.py --zip-path dist/eagc_track1_mvp_source.zip
```

The reproducibility check extracts the source zip into `dist/repro_check/`, verifies that excluded runtime artifacts are absent, checks required readiness documents, and runs the fast test tier in the clean extracted project.

## Docker Package Readiness

Recommended Docker checks:

```powershell
python tools/check_docker_context.py
docker build -t eagc-track1-agent:v0.11 .
docker run --rm eagc-track1-agent:v0.11 python tools/run_test_suite.py --tier docker-smoke
```

The Docker image does not include Qwen model weights. It expects an external OpenAI-compatible vLLM endpoint configured with `QWEN_BASE_URL` and `QWEN_MODEL`. The organizer-provided container/runtime strategy is still not available.
