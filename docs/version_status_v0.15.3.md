# Version Status v0.15.3

## Completed Components

- LocalSim Track 1 MVP with exploration, world-model update, planning, execution, exception recovery, logging, and validators.
- Official-style `Track1ProcedureRunner` for exploration, task reception, planning, execution, and evaluation.
- Visual-local hybrid prototype with multi-frame visual sequence support and evidence-based task uncertainty reporting.
- ALFRED offline adapter with a synthetic ALFRED-like fixture conversion test.
- VirtualHome setup diagnostics and local GPU/vLLM coexistence checks.
- Dockerfile, source package tooling, submission bundle tooling, and clean source package reproducibility checks.
- Decomposed test tiers with per-command runtime reports.

## Validated Tests

- `python tools/run_test_suite.py --tier fast`: deterministic, stable, no real Qwen or real vision.
- `python tools/run_test_suite.py --tier targeted-text --timeout-seconds 300`: minimal real Qwen text endpoint smoke.
- `python tools/run_test_suite.py --tier targeted-local-sim --timeout-seconds 600`: fixed LocalSim real smoke, latest observed runtime approximately 283 seconds.
- ALFRED synthetic fixture conversion: `tests/fixtures/alfred/sample_traj_data.json`.

## Known Limitations

- Official EAGC runtime/API/schema has not been integrated.
- Hidden evaluation environments and official scoring are not available.
- ProcTHOR, Habitat, and AI2-THOR real execution are not validated in the stable path.
- VirtualHome remains diagnostics only; real Windows executable smoke has not succeeded.
- Real ALFRED dataset conversion has not been validated locally.
- No model training, fine-tuning, reinforcement learning, or distillation has been performed.
- Visual-local hybrid is symbolic and does not perform physical manipulation.

## External Dependencies Not Bundled

- Qwen3.6-35B-A3B-NVFP4 model weights are not included.
- The local vLLM endpoint is external to the agent package.
- ALFRED datasets are not included or downloaded automatically.
- VirtualHome repo, simulator executable, Unity assets, scene assets, and generated frames/videos are not included.
- ProcTHOR/Habitat/AI2-THOR assets are not included.

## Recommended Official Submission Contents

- Source package generated by `python tools/package_source.py`.
- Submission bundle generated by `python tools/create_submission_bundle.py`.
- Technical report draft and disclosures under `submission_package/`.
- Dockerfile and Docker documentation.
- Sample world model / episode log outputs from `outputs/demo_snapshot/`, if regenerated locally.
- Pre-submission audit report from `python tools/pre_submission_audit.py`.

## What To Say In The Technical Report

- This is a local Track 1 MVP and evaluation baseline, not an official runtime result.
- The system emphasizes auditable world-model construction, symbolic planning, recovery, validation, and reproducibility.
- LocalSim and visual-local hybrid results are local development evidence only.
- `local_heuristic_score` is not an official score.
- No training has been performed.
- ALFRED support is currently offline trajectory conversion with a synthetic fixture test; real ALFRED data is not redistributed.
